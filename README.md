# 카페 키오스크 프로그램

**"효율적인 카페 운영을 위한 통합 관리 솔루션"**

---

## 📌 프로젝트 개요

### 💡 프로젝트 소개
- **개발 기간**: 2024.10.14 ~ 2024.10.23 (10일)  
- **개발자**: 이현건  
- **프로젝트 주제**:  
  키오스크를 활용한 **카페 운영 관리 프로그램** 개발.  
  Java Swing을 이용해 GUI 구현, 데이터베이스 연동을 통한 기능 제공.  

### 💡 개발 동기
- 사용자 편의성을 높이고, 카페 운영의 효율성을 극대화할 수 있는 시스템을 직접 구현해 보고자 함.
- 실제 카페 운영에서 사용할 수 있는 **주문 관리, 매출 확인, 고객 데이터 관리** 기능을 제공.

## 🛠️ 요구사항 정의 및 주요 기능

### 요구사항 정의
1. **주문 관리**:
   - 고객이 전화번호를 입력하면 데이터가 저장됨.
   - 메뉴를 선택하면 수량, 옵션 입력 후 장바구니에 추가.
   - 결제 후 장바구니 초기화 및 매출 테이블에 저장.

2. **매니저 기능**:
   - 로그인 기능을 통해 접근.
   - 매출 데이터 확인 및 관리.

3. **결제 처리**:
   - **결제 방식**: 신용카드, 현금, 계좌이체.
   - 결제 완료 시 payment 테이블에 데이터 저장.

---

## 🗂️ 데이터베이스 설계

### ERD 구조
- **주요 테이블**:
  - `Client`: 고객 데이터 저장 (전화번호, ID 자동 생성).
  - `Payment`: 결제 데이터 저장.
  - `Menu`: 카테고리별 메뉴 관리.
  - `Manager`: 매니저 로그인 정보 관리.

---

## 🚀 기술 스택 및 사용 기술

### **사용 기술**
1. **Java Swing**: GUI 화면 구성 및 이벤트 처리.
2. **CRUD 구현**:
   - DAO(데이터 등록): 데이터 삽입, 삭제.
   - DTO(데이터 전송): 데이터 구조 정의 및 전송.
3. **디자인 패턴**:
   - **싱글톤 패턴**: DAO 클래스에 적용하여 메모리 효율성 증대.
   - **캡슐화**: DTO 객체에 접근제어자(private)와 Getter/Setter 사용.
4. **컬렉션 프레임워크**:
   - `ArrayList`: DB에서 데이터를 불러와 활용.

## 💻 주요 기능

1. **매니저 로그인**:
   - 로그인 성공 시 매출 확인 화면으로 이동.
   - 로그인 실패 시 에러 메시지 표시.

2. **주문 관리**:
   - 고객 전화번호 입력 후 메뉴 화면 표시.
   - 카테고리별 메뉴(커피, 논커피, 에이드, 디저트) 선택 가능.
   - 상품 선택 시 수량 및 옵션 지정 후 장바구니에 추가.

3. **결제 기능**:
   - 신용카드, 현금, 계좌이체 중 결제 방식 선택.
   - 결제 완료 시 payment 테이블에 데이터 저장 및 장바구니 초기화.

## 🙌 개발 경험 및 소감
이번 프로젝트는 Java를 사용하여 GUI 기반의 애플리케이션을 설계하고 구현하는 좋은 경험이었습니다. 특히, Java Swing을 이용한 화면 구성과 이벤트 처리 과정을 통해 사용자 친화적인 인터페이스를 만드는 방법을 배웠습니다.

또한, DAO, DTO 설계와 같은 객체지향 프로그래밍의 원리를 적용하고, 데이터베이스와의 연동 작업을 직접 경험하면서 CRUD의 중요성을 체감할 수 있었습니다.

개발 과정에서는 특히 싱글톤 패턴과 컬렉션 프레임워크를 활용해 효율적인 데이터 관리 방식을 구현한 점이 기억에 남습니다. 이러한 설계 방식을 실제 프로젝트에 적용해 보니 유지보수성과 코드 가독성을 높이는 데 큰 도움이 되었습니다.

처음에는 작은 부분에서 예상치 못한 오류가 발생하기도 했지만, 디버깅과 테스트를 반복하면서 문제 해결 능력을 키울 수 있었습니다. 짧은 기간 동안 진행된 프로젝트였지만, 초기 기획부터 최종 구현까지 모든 과정을 직접 경험하며 한 단계 성장할 수 있는 값진 시간이었습니다.
